URL

A uniform resource locator (URL), colloquially known as a web address, is a reference to a resource on the World Wide Web. A URL specifies the location of a resource on a computer network and a mechanism for retrieving it. A URL is a specific type of Uniform Resource Identifier (URI), although many people use the two terms interchangeably. A URL is most commonly used to reference a web page (HTTP/HTTPS) but is also used for file transfer (FTP), email (mailto), database access (JDBC), and many other applications.

Most web browsers display the URL of a web page above the page in an address bar. As an example of a web page URL, https://www.example.com/index.html indicates protocol https, hostname www.example.com, and file name index.html.

Parts of a URL

Country-code top-level domain

The country-code top-level domain (ccTLD) is the last part of a URL, such as .com, .org, or .net. It indicates the type of organization or the country of origin of the website. For example, .com is typically used for commercial websites, while .org is used for non-profit organizations.

Domain Name

The domain name is the unique identifier for a website. It is the part of the URL that comes before the top-level domain. For example, in https://www.example.com/index.html, the domain name is "example".

Effective top-level domain

The effective top-level domain (eTLD) is the part of the URL that comes after the domain name. It is used to identify the organization or entity that owns the website. For example, in https://www.example.com/index.html, the effective top-level domain is "com".

Hostname

The hostname is the part of the URL that specifies the location of the server where the resource is hosted. For example, in https://www.example.com/index.html, the hostname is "www.example.com".

Path

The path is the part of the URL that specifies the location of the resource on the server. For example, in https://www.example.com/index.html, the path is "/index.html".

Protocol

The protocol is the part of the URL that specifies the method used to access the resource. For example, in https://www.example.com/index.html, the protocol is "https".
